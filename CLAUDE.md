# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Unity 6 (6000.4.0f1) 2D platformer/vehicle game using URP. The only hand-written script is `Assets/player.cs` — `Assets/InputSystem_Actions.cs` is auto-generated by the Input System package and should not be edited manually.

## Building & Running

There are no CLI build commands — the project must be opened and run via the **Unity Editor**. Open the project in Unity 6, load `Assets/Scenes/SampleScene.unity`, and press Play.

To regenerate `InputSystem_Actions.cs` after changing input bindings, select `Assets/InputSystem_Actions.inputactions` in the Project window and click **Generate C# Class** in the Inspector.

## Architecture

### Player Controller (`Assets/player.cs`)

The player is a **kinematic Rigidbody2D** — Unity's physics solver is bypassed entirely. All movement, gravity, and collision response is handled manually in `Update()`.

**Vertical motion flow (each frame):**
1. Accumulate gravity into `verticalVelocity`
2. Run `UpdateWheelSuspension()` for each wheel to get ground hit data and update wheel local Y positions
3. If grounded and the current frame's fall would reach/pass `landingY`, zero velocity and snap to `landingY`
4. `transform.Translate` by remaining `verticalVelocity * dt`

**Suspension system:** Each wheel raycasts downward from its chassis attachment point (local Y = 0). When the hit is within suspension travel range, the wheel snaps directly to the contact position (no spring animation on contact). When airborne, a spring + damper drives the wheel back to full extension (`-suspensionRestLength` in local space). The grounded boolean is derived per-wheel and ORed for the chassis.

**Key grounding invariant:** `landingY = groundHitY + wheelRadius + suspensionRestLength`. The chassis rests here; the wheel center sits exactly on the surface.

### Input

Uses the **new Unity Input System**. `PlayerInputActions` (auto-generated wrapper) is instantiated in `Awake()`. Only `Player/Move` (Vector2) and `Player/Jump` (Button) are wired up. The input asset also defines Look, Attack, Interact, Crouch, Sprint, Previous, Next — these are unused stubs for future expansion.

### Scene

`SampleScene` contains: Player (with Front Wheel and Rear Wheel as children), two static ground platforms on the `Ground` layer, Main Camera, and a Global Light 2D. Wheel children each have a `CircleCollider2D` whose radius drives suspension calculations.
